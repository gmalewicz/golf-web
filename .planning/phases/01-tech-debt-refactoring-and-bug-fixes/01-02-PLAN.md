In each of the 16 HTML template files listed above, replace all loose equality operators with strict equality:

1. Replace `==` with `===` in template expressions
2. Replace `!=` with `!==` in template expressions

**CRITICAL: Semantic safety for null checks.**
- `x == null` checks both `null` and `undefined`. Changing to `x === null` would miss `undefined`.
- In Angular templates, values are typically initialized (via signals or component properties), so `== null` checks are usually checking for a specific null value.
- However, if you find `== null` or `!= null` patterns, use `=== null || === undefined` or the template-safe equivalent, OR verify the variable is never `undefined` before using `=== null`.
- Most template comparisons in this codebase compare against string literals (e.g., `value == 'STROKE_PLAY'`) or numeric literals (e.g., `value == 0`) where `==` and `===` are semantically identical. Replace these confidently.

Process each file:

The `tournament-results.component.html` template is the only file with legacy `*ngIf` and `*ngFor` directives (49 errors). The Angular schematic `ng generate @angular/core:control-flow` cannot auto-migrate this file because it has duplicate `#showDetails` ng-template names.

Manual migration steps:

1. **Rename duplicate ng-template references first:**
   - Find all `<ng-template #showDetails>` elements in the template
   - Rename duplicates to unique names (e.g., `#showDetails1`, `#showDetails2`, `#showDetails3` etc. — one per section/loop context)
   - Update all corresponding `*ngIf="condition; else showDetails"` references to match the renamed templates

2. **Convert all `*ngIf` to `@if`:**
   ```html
   <!-- Before -->
   <div *ngIf="condition">content</div>
   <div *ngIf="condition; else showDetails1">content</div>
   <ng-template #showDetails1>fallback</ng-template>

   <!-- After -->
   @if (condition) {
     <div>content</div>
   }

   @if (condition) {
     <div>content</div>
   } @else {
     fallback
   }
   ```
   Remove all `<ng-template #showDetailsN>` elements that were only used as `else` targets — their content moves into `@else` blocks.

3. **Convert all `*ngFor` to `@for`:**
   ```html
   <!-- Before -->
   <tr *ngFor="let item of items; let i = index">

   <!-- After -->
   @for (item of items; track item; let i = $index) {
     <tr>
   }
   ```
   Use `track item` if items have identity, or `track $index` if they are primitives.

4. **Remove unused imports** from the component .ts file:
   - Remove `NgIf`, `NgFor`, `NgClass` from the `imports` array if they are no longer needed (check if `[ngClass]` is still used — if so, keep `NgClass`)
   - Add any needed imports for the new syntax
